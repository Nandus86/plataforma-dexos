步骤三：协议测试

添加设备后，可以选择指定设备快速进行协议测试，直观看到通过 Device Gateway 能实现的功能，测试功能是否正常。

测试透传设备功能协议

对于未在协议测试界面列出但属于设备支持的功能，可以用过 Device Gateway 透传设备ISAPI协议测试功能。

本文以添加人脸图片至人脸库为例，介绍测试透传设备功能协议和视频协议的具体步骤。

  1. 单击协议测试 > 通用协议 > 透传设备ISAPI协议。
  2. 通过 Device Gateway 透传设备ISAPI协议，获取人脸库信息，比如FDID。
    1. 单击选择，选中支持添加人脸的相机设备。
    2. 将请求方法设置为GET，并用获取人脸库API的URI /ISAPI/Intelligent/FDLib 替换URL中 <ISAPIURI>。
    3. 单击发送请求。
  3. 获取人脸库信息后，通过视频设备的添加人脸协议，添加人脸到指定人脸库。
    1. 单击视频协议 > 设备操作 > 添加人脸。
    2. 在请求参数的输入框中输入步骤二中获取的FDID。
    3. 单击，上传图片。
    4. 单击发送请求。可在响应内容框中查看上传结果。

    网页端可参看以下动图（PDF版本不支持），直观了解具体步骤。

通用协议测试

对于设备管理、设备操作、网关配置、开启预览等操作，可以通过通用协议测试，测试某一项功能是否可以正常使用。

以测试预览功能为例，介绍协议通用测试的步骤。只有 Firefox 、 Chrome、Edge浏览器 支持预览、回放功能。 网页仅支持 G.711ulaw、G.711alaw 以及 AAC 的音频格式。

  1. 单击协议测试 > 通用协议 > 视频和音频 > 开始预览。
  2. 单击选择，选中支持预览的设备。

    选择设备后，URL中的<uuid>将自动填充。

  3. 可选操作： 若要在ISUP2.0设备开启抓图功能，单击网关配置 > 系统设置 > 报警转发，启用报警转发。
  4. 可选操作： 设置请求超时。
  5. 单击发送请求。
  6. 单击预览。
    图 2 测试预览功能
  7. 可选操作： 执行以下操作。
    表 1 开启预览后的可选操作

    操作

    描述

    开启/暂停预览

    单击/开启或暂停预览。

    开启/停止录像

    单击/开启或停止录像。

    说明

    Firefox浏览器不支持该功能。

    打开/关闭声音

    单击/开启或关闭声音。

    抓拍BMP格式图片

    单击抓拍BMP格式图片。

    抓拍JPEG格式图片

    单击抓拍JPEG图片。

音视频协议测试

支持测试如开启回放、语音对讲等音视频协议测试是否正常。

以测试开始回放、获取对讲通道功能为例，介绍视频协议测试的步骤。只有 Firefox 、 Chrome、Edge浏览器 支持对讲、回放功能。 网页仅支持 G.711ulaw、G.711alaw 以及 AAC 的音频格式。

测试回放功能

  1. 单击协议测试 > 视频协议 > 视频和音频 > 开始回放。
  2. 单击选择，选中支持回放的设备。

    选择设备后，URL中的<uuid>将自动填充。

  3. 可选操作： 设置请求超时。
  4. 单击发送请求。
  5. 设置endTime和startTime。
  6. 单击回放。
    图 3 测试回放功能
  7. 可选操作： 执行以下操作。
    表 2 开启回放后的可选操作

    操作

    描述

    开启/暂停回放

    单击/开启或暂停回放。

    开启/停止录像

    单击/开启或停止录像。

    说明

    Firefox浏览器不支持该功能。

    慢放/快放

    单击/慢放或快放录像。

    打开/关闭声音

    单击/打开或关闭声音。

    选择录像

    在回放页面右上角单击选择录像单击来播放录像。默认播放指定时间内第一条录像。

测试对讲功能

  1. 配置HTTPS。
    1. 安装并创建证书。具体参见配置HTTPS。
    2. 单击网关配置 > 网络 > 端口 > 网关访问端口，查看HTTPS端口状态是否正常。
    3. 在浏览器地址栏输入HTTPS网址，访问Device Gateway。
  2. 测试对讲功能。
    1. 单击协议测试 > 视频协议 > 视频和音频 > 获取对讲通道。
    2. 单击选择，选择支持对讲的设备。
    3. 单击发送请求。
    4. 单击开始/停止语音对讲。
